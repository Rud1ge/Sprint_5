# Sprint 5

UI-автотесты E2E для учебного сервиса объявлений [«Доска»](https://qa-desk.education-services.ru/).

Проект проверяет ключевые пользовательские сценарии в браузере: открытие сайта, взаимодействие с формами и проверка элементов интерфейса после действий. Локаторы вынесены в `tests/locators.py`, общая настройка браузера — в `conftest.py`.

## Регистрация пользователя

`tests/test_user_registration.py` — сценарий регистрации нового пользователя через форму «Вход и регистрация».

1. Нажать «Вход и регистрация».
2. Нажать «Нет аккаунта».
3. Заполнить Email уникальным адресом и нажать «Создать аккаунт».
4. Проверить: на главной странице в правом верхнем углу, рядом с кнопкой «Разместить объявление», отображаются аватар и имя **User.**

**Ожидаемый результат:** регистрация прошла успешно, пользователь авторизован и виден в шапке сайта.

## Регистрация пользователя c email не по маске **\*\*\***@**\*\*\***.\*\*\*

`tests/test_user_registration_not_by_mask.py` — негативный сценарий регистрации с невалидным email.

1. Нажать «Вход и регистрация».
2. Нажать «Нет аккаунта».
3. Заполнить поле Email невалидным значением и нажать «Создать аккаунт».
4. Проверить: поля **Email**, **Пароль**, **Повторите пароль** выделены красным, под полем Email отображается сообщение **«Ошибка»**.

**Ожидаемый результат:** регистрация не выполняется, форма показывает валидационные ошибки.

## Регистрация уже существующего пользователя

`tests/test_user_registration_existing_user.py` — негативный сценарий регистрации пользователя, который уже есть в системе.

1. Нажать «Вход и регистрация».
2. Нажать «Нет аккаунта».
3. Заполнить все поля формы регистрации данными уже существующего пользователя и нажать «Создать аккаунт».
4. Проверить: поля **Email**, **Пароль**, **Повторите пароль** выделены красным, под полем Email отображается сообщение **«Ошибка»**.

**Ожидаемый результат:** повторная регистрация существующего пользователя невозможна, форма показывает валидационные ошибки.

## Login пользователя

`tests/test_user_login.py` — сценарий авторизации уже существующего пользователя.

1. Нажать «Вход и регистрация».
2. Заполнить все поля формы авторизации и нажать «Войти».
3. Проверить: в правом верхнем углу, около кнопки «Разместить объявление», отображаются аватар пользователя и имя **User.**

**Ожидаемый результат:** пользователь успешно авторизован, элементы профиля отображаются в шапке сайта.

## Logout пользователя

`tests/test_user_logout.py` — сценарий выхода из авторизованного аккаунта.

1. Авторизоваться под заранее созданным пользователем.
2. Нажать кнопку «Выйти».
3. Проверить: аватар пользователя и имя **User** больше не отображаются в правом верхнем углу около кнопки «Разместить объявление», там снова отображается кнопка **«Вход и регистрация»**.

**Ожидаемый результат:** пользователь успешно выходит из аккаунта, интерфейс возвращается в состояние неавторизованного пользователя.

## Создание объявления неавторизованным пользователем

`tests/test_create_ad_unauthorized_user.py` — негативный сценарий попытки размещения объявления без авторизации.

1. Нажать кнопку «Разместить объявление».
2. Проверить: отображается модальное окно с заголовком «Чтобы разместить объявление, авторизуйтесь».

**Ожидаемый результат:** создание объявления недоступно без авторизации, отображается модальное окно с требованием войти в аккаунт.

## Создание объявления авторизованным пользователем

`tests/test_create_ad_authorized_user.py` — сценарий успешного создания объявления авторизованным пользователем.

1. Авторизоваться под заранее созданным пользователем.
2. Заполнить поля «Название», «Описание товара», «Стоимость» (стоимость в числовом формате).
3. Выбрать из Dropdown «Категорию» и «Город».
4. Выбрать RadioButton «Состояние товара».
5. Нажать кнопку «Опубликовать».
6. Перейти в профиль пользователя.
7. Проверить: в блоке «Мои объявления» отображается созданное объявление.

**Ожидаемый результат:** объявление успешно публикуется и отображается в блоке «Мои объявления» в профиле пользователя.
